# 🏋️‍♂️ Hybrid Workout Web App

เว็บแอปพลิเคชันแบบ Mobile-First ในธีม Dark Mode สำหรับติดตามและบันทึกความคืบหน้าการออกกำลังกายตามตาราง **Upper/Lower Hybrid + Anterior/Posterior Focus** 

แอปพลิเคชันนี้ออกแบบมาให้ใช้งานง่ายบนโทรศัพท์มือถือ โดยมีฟีเจอร์เด่นคือการปรับเปลี่ยนโหมดตามระดับพลังงานในแต่ละวัน (Full, Lazy, Very Lazy) พร้อมระบบสั่นตอบสนอง (Haptic Feedback) และการบันทึกความคืบหน้าอัตโนมัติ

---

## ✨ ฟีเจอร์หลัก (Key Features)

*   **📱 Mobile-First Design & Dark Mode**: หน้าตาเว็บสวยงามสไตล์พรีเมียม ถนอมสายตา และรองรับการแสดงผลบนหน้าจอมือถือทุกขนาดอย่างสมบูรณ์แบบ
*   **⚡ 3 Intensity Modes (โหมดความหนักตามระดับพลังงาน)**:
    *   **Full Mode**: เล่นเต็มตารางสำหรับวันที่ฟิตเต็มร้อย
    *   **Lazy Mode**: ลดจำนวนท่าเหลือเฉพาะท่าหลัก 3 ท่าสำหรับวันที่มีเวลาจำกัด
    *   **Very Lazy Mode**: เล่นขั้นต่ำ 2 ท่าหลักเพื่อรักษา Momentum และความต่อเนื่อง
*   **💾 Auto-Save (localStorage)**: บันทึกข้อมูลเซ็ตที่เล่นเสร็จแล้วโดยอัตโนมัติ ไม่ต้องกังวลว่าความคืบหน้าจะหายเมื่อปิดหรือรีเฟรชหน้าเว็บ
*   **📳 Haptic Feedback**: ระบบสั่นเตือนเบา ๆ (50ms) เมื่อกดบันทึกเซ็ตที่ทำสำเร็จ (รองรับบนอุปกรณ์มือถือที่รองรับระบบสั่น)
*   **🖼️ Clean Illustrations**: แสดงเฉพาะรูปวาดท่าทางและโมเดลกล้ามเนื้อส่วนที่ใช้งาน (Target Muscles) ช่วยให้ประหยัดพื้นที่หน้าจอและอ่านง่าย

## 📅 รายละเอียดตารางออกกำลังกาย (Workout Split)

ตารางนี้จะแบ่งการเล่นออกเป็น 4 วัน เพื่อกระจายพลังงานและพักฟื้นกล้ามเนื้อได้อย่างมีประสิทธิภาพสูงสุด:

1.  **Day 1: Upper Anterior Focus** (เน้น อก, ไหล่หน้า/ข้าง, หลังแขน และมีท่าดึงช่วยบาลานซ์)
2.  **Day 2: Lower Posterior Focus** (เน้น ก้น, ต้นขาด้านหลัง, หลังล่าง และน่อง)
3.  **Day 3: Upper Posterior Focus** (เน้น หลัง, ไหล่หลัง และหน้าแขน)
4.  **Day 4: Lower Anterior Focus** (เน้น ต้นขาด้านหน้า และแกนกลางลำตัว)

```bash

└── images/          # โฟลเดอร์เก็บภาพประกอบท่าออกกำลังกายที่ถูกตัดแล้ว (dayX_Y.png)
```

## 🚀 วิธีการใช้งาน (Getting Started)

คุณสามารถรันโปรเจกต์นี้ได้ง่าย ๆ 2 วิธี:

### วิธีที่ 1: ดับเบิ้ลคลิกเปิดโดยตรง (Direct Open)
*   ดับเบิ้ลคลิกที่ไฟล์ `index.html` เพื่อเปิดใช้งานผ่านเว็บเบราว์เซอร์บนคอมพิวเตอร์ของคุณได้ทันที

### วิธีที่ 2: รันผ่าน Local Server (แนะนำสำหรับการทดสอบ)
หากต้องการรันเพื่อจำลองการทำงาน สามารถใช้ Python รันเว็บเซิร์ฟเวอร์จำลองได้:

```bash
# สำหรับ Python 3
python -m http.server 8000
```
จากนั้นเปิดเบราว์เซอร์ไปที่ [http://localhost:8000](http://localhost:8000)

## ☁️ การนำขึ้นใช้งานบน Vercel (Deployment & Vercel KV)

โปรเจกต์นี้รองรับการอัปโหลดขึ้นใช้งานบน **Vercel** และเชื่อมต่อกับ **Vercel KV** (Redis) โดยตรงเพื่อเซฟข้อมูลถาวร:

1.  **นำโค้ดขึ้น GitHub**: อัปโหลดโปรเจกต์ของคุณขึ้นบน GitHub Repository ของคุณ
2.  **เชื่อมต่อกับ Vercel**: 
    *   เข้าเว็บ Vercel และ Import โปรเจกต์นี้เข้ามา
    *   Vercel จะตรวจจับหน้าบ้านและสร้าง API `/api/progress` จากฟังก์ชัน Serverless ให้อัตโนมัติ
3.  **สร้างและเชื่อมต่อ Vercel KV**:
    *   ที่แถบเครื่องมือของ Vercel ไปที่เมนู **Storage** และเลือกสร้าง **KV** (Redis Database)
    *   กดเชื่อมต่อ (Connect) ฐานข้อมูล KV นั้นเข้ากับโปรเจกต์เว็บของคุณ
    *   Vercel จะทำการใส่ตัวแปรสภาพแวดล้อม (Environment Variables) เช่น `KV_REST_API_URL` และ `KV_REST_API_TOKEN` ให้เว็บแอปของคุณรันใช้งานได้โดยอัตโนมัติ

*หมายเหตุ: เมื่อเปิดใช้งานผ่านหน้าเว็บเครื่องเครื่องมือพัฒนาในเครื่องตัวเอง (Localhost) ที่ไม่มีตัวแปรสภาพแวดล้อมของ Vercel ระบบจะทำการบันทึกข้อมูลและดึงข้อมูลผ่าน `localStorage` ในเครื่องของคุณเป็นตัวสำรองแทน*

## 🛠️ รายละเอียดการพัฒนาภาพประกอบ (Illustration Optimization)
ภาพประกอบท่าออกกำลังกายเดิมที่เป็นข้อมูลรูปภาพการ์ดขนาดใหญ่ (มีหัวข้อ ตัวหนังสือ และรายละเอียดปนมา) ได้ถูกตัดแยก (Crop) เอาเฉพาะภาพสาธิตการเคลื่อนไหวและแผนผังกล้ามเนื้อตรงส่วนกลางของการ์ด เพื่อให้แอปพลิเคชันดูคลีน อ่านง่าย และเหมาะสมกับการแสดงผลบนมือถือ
